🏗️ 1. 전체 아키텍처 한눈에 보기

Spring Boot 레이어드 아키텍처
┌────────────────────────────────────────────────────┐
│                 사용자 (브라우저)                      │
└──────────────────────┬─────────────────────────────┘
                       ↓ HTTP 요청 (GET /tour/list)
┌────────────────────────────────────────────────────┐
│             Interceptor (인증/인가 체크)                │
│    LoginCheckInterceptor → AdminCheckInterceptor    │
└──────────────────────┬─────────────────────────────┘
                       ↓ preHandle() = true면 통과
┌────────────────────────────────────────────────────┐
│              Controller 레이어                       │
│    @Controller + @GetMapping + @PostMapping         │
│    → HTTP 요청/응답 처리, Model 구성                  │
└──────────────────────┬─────────────────────────────┘
                       ↓ tourService.findAll() 호출
┌────────────────────────────────────────────────────┐
│              Service 레이어                          │
│    @Service + @Transactional                        │
│    → 비즈니스 로직, DTO 변환, 파일 처리                 │
└──────────────────────┬─────────────────────────────┘
                       ↓ tourRepository.findAll() 호출
┌────────────────────────────────────────────────────┐
│              Repository 레이어                       │
│    JpaRepository → Spring Data JPA                  │
│    → 메서드명 쿼리, @Query, JPQL                      │
└──────────────────────┬─────────────────────────────┘
                       ↓ SQL 자동 생성 실행
┌────────────────────────────────────────────────────┐
│              Database (PostgreSQL)                  │
│    tour, member, review, boards 테이블              │
└────────────────────────────────────────────────────┘

🔄 2. "관광지 등록" 요청 처리 전체 흐름

  1. 사용자: GET /tour/new 접속

    브라우저에서 관광지 등록 폼 페이지 요청

  2. Interceptor: LoginCheckInterceptor 실행

    세션에 loginMember가 있는지 확인. 없으면 /login으로 리다이렉트. 있으면 계속 진행

  3. Controller: TourController.tourForm() 실행

    model.addAttribute("tourFormDto", new TourFormDto())로 빈 DTO를 HTML에 전달

  4. View: templates/tour/form.html 렌더링

    Thymeleaf가 HTML에 데이터를 채워 응답. 사용자에게 폼 화면 표시

  5. 사용자: 폼 입력 후 POST /tour/new 전송

    이름, 카테고리, 이미지 파일 등을 입력하고 저장 버튼 클릭

  6. Controller: write(@Valid TourFormDto dto) 실행

    @Valid로 입력값 검증. 오류 있으면 폼으로 돌아가기. 없으면 Service 호출

  7. Service: TourService.save(dto) 실행

    FileUploadUtil로 이미지 저장, Builder로 Tour Entity 생성

  8. Repository: tourRepository.save(tour) 실행

    JPA가 SQL INSERT 쿼리 자동 생성 실행, DB에 데이터 저장

  9. Controller: "redirect:/tour/list" 반환

    저장 성공 후 목록 페이지로 리다이렉트 (PRG 패턴)

📋 4. 핵심 어노테이션 전체 정리

어노테이션위치역할
@SpringBootApplicationMain 클래스자동 설정 + 컴포넌트 스캔 활성화
@EntityDomain 클래스JPA 엔티티 선언 (DB 테이블 매핑)
@TableDomain 클래스테이블 이름 지정
@Id + @GeneratedValueDomain 필드기본키 + 자동 증가
@ColumnDomain 필드컬럼 속성 설정
@OneToMany / @ManyToOneDomain 필드테이블 관계 설정
@MappedSuperclassBaseEntity 클래스공통 필드 상속용 부모 클래스
@ServiceService 클래스비즈니스 로직 Bean 등록
@TransactionalService 메서드/클래스트랜잭션 경계 설정
@ControllerController 클래스웹 요청 처리 Bean 등록
@RequestMappingController 클래스기본 URL 경로 설정
@GetMapping / @PostMappingController 메서드HTTP GET/POST 처리
@PathVariable메서드 파라미터URL 경로 변수 바인딩
@RequestParam메서드 파라미터쿼리 파라미터 바인딩
@ModelAttribute메서드 파라미터폼 데이터 → DTO 바인딩
@Valid메서드 파라미터Bean Validation 활성화
@ResponseBodyController 메서드데이터 직접 반환 (AJAX)
@ConfigurationConfig 클래스Spring 설정 클래스 선언
@BeanConfig 메서드반환 객체를 Bean으로 등록
@ValueConfig 필드application.yml 값 주입
@Getter / @Setter클래스/필드Lombok getter/setter 자동 생성
@Builder클래스Lombok 빌더 패턴 자동 생성
@Slf4j클래스Lombok 로그 필드 자동 생성

🚨 5. 보안 이슈 정리

🔴

Tour 프로젝트의 보안 문제점

  • 평문 비밀번호: DB 해킹 시 모든 비밀번호 노출. → BCrypt 사용 필요
  • ddl-auto: create: 재시작 시 모든 데이터 삭제. → update 또는 none 사용
  • 인터셉터 없음: 인증 없이 관리 기능 접근 가능. → LoginCheckInterceptor 추가 필요
✅

Demo 프로젝트가 개선한 것

  • BCrypt 비밀번호 암호화 적용
  • LoginCheckInterceptor + AdminCheckInterceptor 적용
  • ddl-auto: update 사용으로 데이터 보존
  • 세션 http-only: true로 XSS 방어
  • 세션 타임아웃 30분 설정

🚀 6. 다음 학습 방향

  1. Spring Security

    Interceptor보다 더 강력한 인증/인가 프레임워크. JWT 토큰 인증도 배울 수 있습니다.

    전역 예외 처리로 사용자 친화적인 오류 페이지를 만드는 방법을 배우세요.

    단위 테스트와 통합 테스트를 작성하는 법을 배우면 더 안정적인 코드를 작성할 수 있습니다.

🏆 7. 최종 요약 — 핵심 법칙

  • 🌱 진입점: @SpringBootApplication → main() 하나로 서버 시작
  • 🗃️ Entity: @Entity로 DB 테이블 매핑, @Id로 PK 설정
  • 🔍 Repository: JpaRepository 상속만으로 CRUD 완성
  • ⚙️ Service: @Service + @Transactional로 비즈니스 로직 처리
  • 🌐 Controller: @GetMapping/@PostMapping으로 URL 매핑
  • 📦 DTO: Entity 직접 노출 금지, Form DTO로 받고 Response DTO로 전달
  • 🔒 보안: 비밀번호는 BCrypt, 접근 제어는 Interceptor
  • 🖥️ 뷰: Thymeleaf th:text, th:each, th:if 활용
  • 🪄 Lombok: @Builder로 객체 생성, @RequiredArgsConstructor로 주입
