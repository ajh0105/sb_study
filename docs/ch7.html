🔒 설정 & 보안

@Configuration, @Bean으로 Spring 설정을 작성하는 방법, BCrypt 암호화, Interceptor 인증/인가 패턴을 학습합니다.

⏱️ 예상 학습 시간: 50분 📝 핵심 클래스: 4개 🎯 퀴즈: 50문제

⚙️ 1. @Configuration & @Bean

@Configuration

이 클래스가 Spring 설정 클래스임을 선언합니다. 내부의 @Bean 메서드들이 Spring 컨테이너에 등록됩니다.

@Bean

메서드가 반환하는 객체를 Spring 컨테이너에 Bean으로 등록합니다. 다른 클래스에서 @Autowired나 생성자 주입으로 사용 가능합니다.

@Value("${key}")

application.yml의 값을 가져옵니다.
예: @Value("${file.upload-dir}")는 yml의 file.upload-dir 값을 필드에 주입합니다.

@EnableJpaAuditing

JPA 감사(Auditing) 기능 활성화. @CreatedDate 등 자동 시간 기록이 동작하려면 이 설정이 필요합니다.

FileConfig.java
@Configuration
@Getter

}

    // 내용 없이 어노테이션만으로 기능 활성화
}

🔐 3. BCrypt 암호화 (PasswordConfig)

PasswordConfig.java (demo 프로젝트)
@Configuration
public class PasswordConfig {

    @Bean   // BCryptPasswordEncoder를 Spring Bean으로 등록
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();  // 가장 표준적인 암호화
    }
}
BCrypt 사용 방법
MemberService.java (demo) — BCrypt 활용
@Service
@RequiredArgsConstructor
public class MemberService {

    private final MemberRepository memberRepository;
    private final PasswordEncoder passwordEncoder;  // Bean 주입

    // 회원 가입: 비밀번호 암호화 후 저장
    public void join(MemberFormDto dto) {
        Member member = Member.builder()
            .email(dto.getEmail())
            .password(passwordEncoder.encode(dto.getPassword()))  // ✅ 암호화
            .role(Role.USER)
            .build();
        memberRepository.save(member);
    }

    // 로그인: BCrypt로 검증
    public Member login(LoginDto dto) {
        Member member = memberRepository.findByEmail(dto.getEmail())
            .orElseThrow(() -> new IllegalArgumentException("이메일 없음"));

        // passwordEncoder.matches(입력 비밀번호, DB에 저장된 해시)
        if (!passwordEncoder.matches(dto.getPassword(), member.getPassword())) {
            throw new IllegalArgumentException("비밀번호 불일치");
        }
        return member;
    }
}
🚨

BCrypt가 중요한 이유

DB가 해킹당해도 BCrypt 해시는 원본 비밀번호 복원이 거의 불가능합니다.
tour 프로젝트: password를 평문 저장 → 보안 취약
demo 프로젝트: BCrypt로 암호화 후 저장 → 올바른 방식
실제 서비스에서는 반드시 demo 프로젝트 방식을 사용해야 합니다!

🛡️ 4. Interceptor — 인증/인가 처리

Interceptor는 Controller 실행 전후에 공통 처리를 삽입하는 패턴입니다. 로그인 확인, 권한 확인 등에 사용됩니다.

LoginCheckInterceptor.java
@Slf4j   // log.info() 등 로깅 메서드 자동 생성
public class LoginCheckInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                              HttpServletResponse response,
                              Object handler) throws Exception {

        String requestURI = request.getRequestURI();  // 요청 URL

        // 세션 가져오기 (false = 새 세션 생성 안 함)
        HttpSession session = request.getSession(false);

        // 세션 없거나 로그인 정보 없으면 로그인 페이지로 이동
        if (session == null || session.getAttribute("loginMember") == null) {
            log.info("미인증 사용자 요청 거부: {}", requestURI);
            response.sendRedirect("/login?redirectURL=" + requestURI);
            return false;  // ← false 반환 = 컨트롤러 실행 차단!
        }

        return true;  // ← true 반환 = 컨트롤러 정상 실행
    }
}
AdminCheckInterceptor.java
@Slf4j
public class AdminCheckInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest req, HttpServletResponse res, Object handler) throws Exception {
        HttpSession session = req.getSession(false);
        if (session == null) {
            res.sendRedirect("/login");
            return false;
        }
        Member member = (Member) session.getAttribute("loginMember");
        if (member == null || member.getRole() != Role.ADMIN) {  // 관리자 확인
            res.sendRedirect("/login");
            return false;
        }
        return true;
    }
}

🔧 5. WebConfig — 인터셉터 등록

WebConfig.java
@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {

        // 로그인 체크 인터셉터
        registry.addInterceptor(new LoginCheckInterceptor())
                .order(1)                    // 실행 순서
                .addPathPatterns("/**")        // 모든 URL에 적용
                .excludePathPatterns(          // 예외: 로그인 없이 접근 가능
                        "/", "/index", "/login", "/signup", "/logout",
                        "/check-email",
                        "/css/**", "/js/**", "/images/**"  // 정적 파일
                );

        // 관리자 전용 인터셉터
        registry.addInterceptor(new AdminCheckInterceptor())
                .order(2)
                .addPathPatterns("/admin/**");  // /admin/... URL만 적용
    }
}
💡

인터셉터 동작 원리

  • order(1): 낮을수록 먼저 실행
  • addPathPatterns("/**"): 모든 URL에 적용
  • excludePathPatterns(): 인터셉터를 적용하지 않을 URL 목록
  • preHandle() 반환값이 false이면 컨트롤러 실행 안 됨

🍪 6. 세션 보안 설정 (application.yml)

application.yml (demo 프로젝트)
server:
  servlet:
    session:
      timeout: 30m          # 세션 유효 시간 30분
      cookie:
        name: BOARDSESSION  # 세션 쿠키 이름
        http-only: true      # JS에서 쿠키 접근 불가 (XSS 방어)
timeout: 30m

마지막 요청 후 30분 동안 활동이 없으면 세션이 자동 만료됩니다. 자동 로그아웃 효과입니다.

http-only: true

JavaScript에서 쿠키에 접근할 수 없게 합니다. XSS(Cross-Site Scripting) 공격으로 세션을 탈취하는 것을 방어합니다.

✅ 7. 챕터 핵심 요약

  • @Configuration: Spring 설정 클래스 선언
  • @Bean: 메서드 반환 객체를 Spring 컨테이너에 등록
  • @Value("${yml.key}"): application.yml 값 주입
  • BCryptPasswordEncoder: 단방향 암호화. 실제 서비스 필수!
  • HandlerInterceptor: 컨트롤러 실행 전/후 처리. 로그인/권한 확인에 사용
  • preHandle(): true 반환 → 계속 실행, false 반환 → 차단
  • WebMvcConfigurer: 인터셉터 등록, URL 패턴 설정
