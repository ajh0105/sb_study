🗃️ Entity / 도메인

데이터베이스 테이블을 Java 클래스로 표현하는 방법 (JPA Entity)을 배웁니다. 어노테이션 한 줄 한 줄의 의미를 이해합니다.

⏱️ 예상 학습 시간: 50분 📝 핵심 어노테이션: 12개 🎯 퀴즈: 50문제

❓ 1. JPA란 무엇인가?

JPA (Java Persistence API)는 Java에서 데이터베이스를 쉽게 다룰 수 있게 해주는 표준 기술입니다.


기존에는 SQL을 직접 작성해야 했지만, JPA를 사용하면 Java 클래스(Entity)와 DB 테이블을 연결하여 SQL 없이도 DB 조작이 가능합니다.

Java 클래스 (Entity)
↔
JPA (Hibernate)
↔
DB 테이블
💡

Hibernate = JPA 구현체

JPA는 표준 인터페이스이고, Hibernate가 실제 구현체입니다. Spring Boot에서 spring-boot-starter-data-jpa를 추가하면 Hibernate가 자동으로 사용됩니다.

🏷️ 2. @Entity 핵심 어노테이션

Tour.java — 관광지 엔티티 전체 코드
package com.example.tour.domain;

import jakarta.persistence.*;
import lombok.*;
import java.util.ArrayList;
import java.util.List;

@Entity                  // ① 이 클래스가 DB 테이블과 연결됨을 선언
@Table(name = "tour")   // ② DB 테이블 이름 지정 (기본값: 클래스명 소문자)
@Getter                  // ③ Lombok: 모든 필드의 getter 자동 생성
@Setter                  // ④ Lombok: 모든 필드의 setter 자동 생성
@NoArgsConstructor       // ⑤ Lombok: 기본 생성자 자동 생성 (JPA 필수!)
@AllArgsConstructor      // ⑥ Lombok: 모든 필드를 인수로 받는 생성자
@Builder                 // ⑦ Lombok: 빌더 패턴 자동 생성
public class Tour extends BaseEntity {  // ⑧ BaseEntity 상속 (createdAt 포함)

    @Id                                    // ⑨ 기본키(PK) 필드
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // ⑩ AUTO INCREMENT
    private Long id;

    @Enumerated(EnumType.STRING)          // ⑪ Enum을 문자열로 저장
    @Column(nullable = false)
    private TourCategory category;

    @Column(nullable = false, length = 100)  // ⑫ NOT NULL, 최대 100자
    private String name;

    private String thumb;      // 썸네일 이미지 파일명
    private String address;
    private String tel;

    @Column(columnDefinition = "TEXT")   // TEXT 타입으로 저장
    private String comment;

    @Column(name = "traffic_map")        // DB 컬럼명 직접 지정
    private String trafficMap;

    @Column(name = "tour_map")
    private String tourMap;

    @Column(name = "food_map")
    private String foodMap;

    @Column(name = "bed_map")
    private String bedMap;

    // Tour와 Review의 관계: 한 관광지에 여러 리뷰 (1:N)
    @OneToMany(mappedBy = "tour",           // Review.tour 필드가 관계의 주인
            cascade = CascadeType.ALL,       // Tour 삭제 시 Review도 삭제
            orphanRemoval = true)            // 관계 끊기면 고아 객체 삭제
    private List<Review> reviews = new ArrayList<>();
}
@Entity

이 Java 클래스가 JPA 엔티티임을 선언합니다. JPA가 이 클래스를 DB 테이블과 연결하여 관리합니다.

@Table(name = "tour")

매핑할 DB 테이블 이름을 지정합니다. 생략하면 클래스명을 소문자로 변환한 이름이 사용됩니다.

@Id

해당 필드가 기본키(Primary Key)임을 나타냅니다. 모든 Entity는 반드시 @Id 필드가 있어야 합니다.

@GeneratedValue

기본키 자동 생성 전략 설정. IDENTITY는 DB의 AUTO_INCREMENT를 사용합니다. (1, 2, 3...)

@Column

필드와 DB 컬럼을 매핑. nullable=false, length=100, name="컬럼명" 등의 옵션 설정 가능

@Enumerated(STRING)

Enum 타입을 DB에 저장할 방식 지정. STRING은 "FOOD", "CULTURE" 같은 문자열로 저장. ORDINAL은 0,1,2 숫자로 저장 (비권장)

🏗️ 3. BaseEntity — 공통 필드 상속

여러 Entity에서 공통으로 사용하는 필드(예: 생성일시)를 한 곳에 모아서 상속으로 재사용합니다.

BaseEntity.java
@Getter
@MappedSuperclass   // ← 이 클래스 자체는 테이블이 아닌 부모 클래스임을 선언
public abstract class BaseEntity {

    @CreatedDate
    @Column(name = "created_at", nullable = false)
    private LocalDateTime createdAt;

    @PrePersist   // ← DB에 저장되기 전에 자동 실행되는 메서드
    public void prePersist() {
        this.createdAt = LocalDateTime.now();  // 현재 시간 자동 저장
    }
}
@MappedSuperclass

JPA에서 부모 클래스의 필드를 자식 클래스의 테이블에 포함시킵니다. BaseEntity 자체는 테이블이 만들어지지 않습니다.

@PrePersist

JPA가 Entity를 DB에 저장하기 직전에 자동으로 호출됩니다. 생성 시간 자동 설정에 활용합니다.

🔗 4. 관계 매핑 (Relationship Mapping)

① @OneToMany — 1:N 관계 (Tour ↔ Review)

하나의 Tour(관광지)에 여러 개의 Review(리뷰)가 달릴 수 있습니다.

Tour.java — 1:N 관계
// Tour 클래스 (1 쪽)
@OneToMany(
    mappedBy = "tour",        // Review 클래스의 'tour' 필드가 FK를 관리
    cascade = CascadeType.ALL, // Tour 삭제 시 연관된 Review도 함께 삭제
    orphanRemoval = true       // reviews 목록에서 제거되면 Review DB도 삭제
)
private List<Review> reviews = new ArrayList<>();
② @ManyToOne — N:1 관계 (Review ↔ Tour)
Review.java — N:1 관계
// Review 클래스 (N 쪽) — DB에 실제 외래키(FK) 컬럼이 생성됨
@ManyToOne(fetch = FetchType.LAZY)   // 지연 로딩: Review를 조회할 때 Tour는 아직 로딩 안 함
@JoinColumn(name = "tour_id", nullable = false)  // DB 컬럼명: tour_id
private Tour tour;

@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "member_id", nullable = false)
private Member member;
어노테이션관계예시FK 위치
@OneToMany1:NTour(1) : Review(N)N 쪽
@ManyToOneN:1Review(N) : Tour(1)N 쪽 (이 쪽에 FK 컬럼 생성)
@OneToOne1:1User(1) : Profile(1)한쪽에
@ManyToManyN:MStudent(N) : Course(M)중간 테이블
fetch = FetchType.LAZY

지연 로딩: Review를 조회할 때 연관된 Tour는 즉시 DB에서 가져오지 않습니다. 실제로 review.getTour()를 호출할 때 쿼리가 실행됩니다. 성능 최적화에 중요!

fetch = FetchType.EAGER

즉시 로딩: 연관 객체를 항상 함께 로딩합니다. 불필요한 쿼리가 많이 실행될 수 있어 일반적으로 LAZY를 권장합니다.

cascade = ALL

부모 엔티티의 상태 변화를 자식에게 전파. ALL = 저장,수정,삭제,병합 모두 전파. Tour를 삭제하면 그 Tour의 모든 Review도 자동 삭제됩니다.

@JoinColumn

FK(외래 키) 컬럼 이름을 지정합니다. name="tour_id"이면 review 테이블에 tour_id 컬럼이 생성됩니다.

👤 5. Member & Review 엔티티

Member.java
Member.java
@Entity
@Table(name = "member")
@Getter @Setter
@NoArgsConstructor @AllArgsConstructor @Builder
public class Member extends BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true, length = 50)  // unique=true: 중복 불가
    private String username;

    @Column(nullable = false)
    private String password;

    @Column(nullable = false, length = 50)
    private String name;

    @Column(unique = true)  // 이메일도 중복 불가
    private String email;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private MemberRole role = MemberRole.USER;  // 기본값: USER
}
Review.java
Review.java
@Entity
@Table(name = "review")
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class Review extends BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "tour_id", nullable = false)
    private Tour tour;   // 어떤 관광지의 리뷰인지

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "member_id", nullable = false)
    private Member member;  // 누가 쓴 리뷰인지

    @Column(nullable = false, columnDefinition = "TEXT")
    private String comment;

    @Column(name = "star_rate", nullable = false)
    private Double starRate;   // 별점 1~5
}

🏷️ 6. Enum 타입

Enum은 미리 정해진 상수 목록을 표현합니다. DB에 문자열로 저장되어 가독성이 좋습니다.

TourCategory.java & MemberRole.java
// 관광지 카테고리
public enum TourCategory {
    FOOD,      // 음식점
    STAY,      // 숙박
    CULTURE,   // 문화
    NATURE     // 자연
}

// 회원 권한
public enum MemberRole {
    USER,   // 일반 사용자
    ADMIN   // 관리자
}

// Demo 프로젝트 게시판 타입
public enum BoardType {
    FREE,    // 자유게시판
    QNA,     // Q&A 게시판
    REPORT   // 관광불편신고
}
💡

왜 Enum을 쓸까?

문자열 직접 사용 시: "FOOD", "food", "음식점" 등 오타 위험
Enum 사용 시: TourCategory.FOOD로 타입 안전하게 사용 가능. 컴파일 시 오류 발견 가능!

✅ 7. 챕터 핵심 요약

  • JPA = Java 클래스로 DB를 다루는 기술
  • @Entity + @Table: 클래스를 DB 테이블과 연결
  • @Id + @GeneratedValue(IDENTITY): AUTO INCREMENT 기본키
  • @Column: 컬럼 속성 설정 (nullable, length, name...)
  • @Enumerated(STRING): Enum을 문자열로 DB에 저장
  • @OneToMany + @ManyToOne: 테이블 간 관계 표현
  • FetchType.LAZY: 성능을 위한 지연 로딩 사용 권장
  • @MappedSuperclass: 공통 필드(createdAt)를 부모 클래스로 분리
