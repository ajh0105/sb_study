🌐 Controller 계층

사용자의 HTTP 요청을 받아 처리하고 응답을 돌려주는 Controller를 완전히 이해합니다. 어노테이션과 파라미터 바인딩까지 상세 학습!

⏱️ 예상 학습 시간: 50분 📝 핵심 어노테이션: 10개+ 🎯 퀴즈: 50문제

❓ 1. Controller란?

Controller는 사용자가 브라우저에서 특정 URL로 접속할 때 어떤 처리를 할지 결정합니다.

@Controller

HTML 뷰(Thymeleaf)를 반환하는 컨트롤러. 메서드가 반환하는 String은 HTML 파일 경로입니다.
예: return "tour/list" → templates/tour/list.html

@RestController

JSON/XML 데이터를 반환하는 컨트롤러. REST API 개발에 사용됩니다. @Controller + @ResponseBody 합본. 이 프로젝트는 @Controller 사용

@RequestMapping

클래스 레벨에서 기본 URL 경로를 설정합니다.
@RequestMapping("/tour")이면 이 Controller의 모든 URL은 /tour로 시작합니다.

@ResponseBody

메서드 반환값을 HTTP 응답 body에 직접 넣습니다. AJAX용 응답(JSON, String, boolean 등)을 반환할 때 사용합니다.

🏖️ 2. TourController 전체 분석

TourController.java — 완전 분석
@Controller
@RequiredArgsConstructor
@RequestMapping("/tour")   // 모든 URL이 /tour 로 시작
public class TourController {

    private final TourService tourService;
    private final ReviewService reviewService;

    // GET /tour/list → 관광지 목록 페이지
    @GetMapping("/list")
    public String list(Model model) {
        model.addAttribute("list", tourService.findAll());  // HTML에 데이터 전달
        return "tour/list";  // templates/tour/list.html
    }

    // GET /tour/detail/{id} → 상세 페이지
    @GetMapping("/detail/{id}")
    public String detail(@PathVariable("id") Long id, Model model) {
        Tour tour = tourService.findById(id);
        model.addAttribute("tour", tour);
        model.addAttribute("reviews", reviewService.findByTour(id));
        model.addAttribute("reviewForm", new ReviewFormDto());
        return "tour/detail";
    }

    // GET /tour/new → 등록 폼 화면
    @GetMapping("/new")
    public String tourForm(Model model) {
        model.addAttribute("tourFormDto", new TourFormDto());  // 빈 폼 객체
        model.addAttribute("categories", TourCategory.values());  // 카테고리 목록
        return "tour/form";
    }

    // POST /tour/new → 등록 처리
    @PostMapping("/new")
    public String write(@Valid @ModelAttribute TourFormDto dto,
                        BindingResult bindingResult,   // 검증 결과
                        Model model) {
        if (bindingResult.hasErrors()) {       // 검증 오류 있으면 폼으로 돌아가기
            model.addAttribute("categories", TourCategory.values());
            return "tour/form";
        }
        try {
            tourService.save(dto);
        } catch (Exception e) {
            e.printStackTrace();
        }
        return "redirect:/tour/list";  // 저장 후 목록으로 리다이렉트
    }

    // GET /tour/delete/{id} → 삭제
    @GetMapping("/delete/{id}")
    public String delete(@PathVariable("id") Long id) {
        tourService.delete(id);
        return "redirect:/tour/list";
    }
}

🏷️ 3. 주요 어노테이션 & 매개변수

@GetMapping("/path")

HTTP GET 요청을 처리합니다. 브라우저에서 URL을 직접 입력하거나 링크를 클릭할 때 GET 요청이 발생합니다.

@PostMapping("/path")

HTTP POST 요청을 처리합니다. HTML 폼의 method="post"로 데이터를 전송할 때 사용됩니다.

@PathVariable

URL 경로의 변수를 메서드 파라미터로 받습니다.
예: /tour/detail/{id}에서 {id} 값을 @PathVariable Long id로 받습니다.

@RequestParam

URL의 쿼리 파라미터를 받습니다.
예: /check-username?username=hong에서 @RequestParam String username으로 "hong"을 받습니다.

@ModelAttribute

HTML 폼 데이터를 DTO 객체로 자동 바인딩합니다. 폼의 name 속성과 DTO 필드명이 일치해야 합니다.

@Valid

DTO의 Bean Validation 어노테이션(@NotBlank 등)을 활성화합니다. 검증 결과는 BindingResult에 저장됩니다.

파라미터역할예시
Model modelHTML에 데이터 전달model.addAttribute("key", value)
HttpSession session세션 데이터 접근session.setAttribute("loginMember", member)
BindingResult검증 결과 저장bindingResult.hasErrors()
@PathVariableURL 경로 변수/tour/{id} → @PathVariable Long id
@RequestParam쿼리 파라미터?name=value → @RequestParam String name

👤 4. MemberController — 세션 & AJAX

MemberController.java — 로그인/로그아웃/AJAX
// 로그인 처리
@PostMapping("/login")
public String login(@ModelAttribute LoginDto dto,
                    HttpSession session,    // 서버에 데이터를 유지하는 세션
                    Model model) {
    try {
        Member member = memberService.login(dto.getUsername(), dto.getPassword());
        session.setAttribute("loginMember", member);  // 세션에 로그인 정보 저장
        return "redirect:/";
    } catch (Exception e) {
        model.addAttribute("error", e.getMessage());  // 에러 메시지 전달
        return "member/login";  // 로그인 폼으로 돌아가기
    }
}

// 로그아웃
@GetMapping("/logout")
public String logout(HttpSession session) {
    session.invalidate();  // 세션 전체 삭제
    return "redirect:/";
}

// AJAX 아이디 중복 체크 — JSON/단순 값 반환
@GetMapping("/check-username")
@ResponseBody   // HTML이 아닌 데이터 직접 반환
public boolean checkUsername(@RequestParam String username) {
    return !memberService.existsByUsername(username);  // true = 사용 가능
}
HttpSession

서버가 브라우저별로 관리하는 임시 저장소입니다. 로그인 정보를 세션에 저장하면 다음 요청에서도 유지됩니다. 브라우저 종료 또는 로그아웃 시 삭제됩니다.

redirect:/path

"redirect:..."를 반환하면 브라우저를 해당 URL로 리다이렉트합니다. 폼 처리 후 새로고침 문제(중복 제출)를 방지하기 위해 사용합니다 (PRG 패턴).

✅ 7. 챕터 핵심 요약

  • @Controller: HTML 뷰 반환 컨트롤러
  • @RequestMapping("/path"): 클래스 레벨 URL 기본 경로
  • @GetMapping / @PostMapping: HTTP GET/POST 처리
  • @PathVariable: URL 경로의 변수 값 받기 /{id}
  • @RequestParam: 쿼리 파라미터 받기 ?name=value
  • Model.addAttribute(): HTML 템플릿에 데이터 전달
  • HttpSession: 로그인 정보 등 서버 임시 저장
  • return "redirect:/path": 다른 URL로 리다이렉트
  • @ResponseBody: JSON/단순 값 직접 반환 (AJAX)
