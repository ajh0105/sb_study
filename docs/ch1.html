🌱 Spring Boot 기초

Spring Boot가 무엇인지, 프로젝트 구조는 어떻게 되는지, 핵심 어노테이션은 무엇인지 처음부터 차근차근 배웁니다.

⏱️ 예상 학습 시간: 40분 📝 핵심 어노테이션: 3개 🎯 퀴즈: 50문제

❓ 1. Spring Boot란 무엇인가?

Spring Boot는 Java로 웹 애플리케이션을 만들기 위한 프레임워크입니다.


복잡한 설정 없이 빠르게 웹 서비스를 만들 수 있도록 도와줍니다. 마치 레고 블록처럼, 필요한 기능을 골라서 조립하면 애플리케이션이 완성됩니다.

💡

Spring vs Spring Boot

Spring: 강력하지만 설정이 매우 복잡한 Java 프레임워크
Spring Boot: Spring을 더 쉽게 사용할 수 있게 만든 것. 자동 설정(Auto Configuration)으로 개발자가 직접 설정할 것을 최소화함

사용자 (브라우저)
→
Controller
→
Service
→
Repository
→
Database

Spring Boot의 기본 처리 흐름 (MVC 패턴)

📁 2. 프로젝트 구조 이해하기

이 프로젝트의 두 앱(tour, demo)은 모두 아래와 같은 구조를 가집니다:

📁 프로젝트 구조 
tour/
├── src/
│   ├── main/
│   │   ├── java/com/example/tour/
│   │   │   ├── TourApplication.java   ← ✅ 실행 진입점 (main 메서드)
│   │   │   ├── config/                ← 설정 클래스들
│   │   │   ├── controller/            ← 웹 요청 처리
│   │   │   ├── domain/ (또는 entity/) ← DB 테이블과 매핑되는 클래스
│   │   │   ├── dto/                   ← 데이터 전송 객체
│   │   │   ├── repository/            ← DB 접근 인터페이스
│   │   │   ├── service/               ← 비즈니스 로직
│   │   │   └── util/                  ← 유틸리티 클래스
│   │   └── resources/
│   │       ├── application.yml        ← ✅ 설정 파일
│   │       └── templates/             ← Thymeleaf HTML 파일들
│   └── test/                          ← 테스트 코드
└── build.gradle                       ← ✅ 빌드 설정 (의존성 관리)
🗂️

패키지 이름의 의미

  • com.example.tour → 회사명.프로젝트명 형태의 패키지명
  • controller → 사용자 요청을 받는 계층
  • service → 실제 업무 처리 계층
  • repository → 데이터베이스와 통신하는 계층
  • domain/entity → 데이터베이스 테이블을 표현하는 클래스

🚀 3. 메인 클래스 분석

Spring Boot 앱이 시작되는 진입점입니다. 모든 Spring Boot 프로젝트는 이 파일에서 시작됩니다.

TourApplication.java 
package com.example.tour;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication          // ① 핵심 어노테이션
public class TourApplication {

    public static void main(String[] args) {
        SpringApplication.run(TourApplication.class, args);  // ② 앱 실행
    }

}
@SpringBootApplication

세 가지 어노테이션을 합친 것:
• @SpringBootConfiguration: 설정 클래스임을 선언
• @EnableAutoConfiguration: 자동 설정 활성화
• @ComponentScan: 이 패키지의 모든 클래스를 Spring이 관리

SpringApplication.run()

Spring Boot 애플리케이션을 실행합니다.
첫 번째 인수: 현재 클래스
두 번째 인수: 커맨드라인 인수
이 메서드 하나로 내장 톰캣 서버가 시작됩니다!

🏠

내장 서버(Embedded Server)

일반적인 Java 웹앱은 톰캣(Tomcat) 서버를 따로 설치해야 합니다. Spring Boot는 톰캣이 앱 안에 내장되어 있어서, main() 메서드만 실행하면 서버가 바로 시작됩니다!

🔧 4. build.gradle 분석

build.gradle은 프로젝트의 설계도입니다. 어떤 라이브러리를 사용할지, Java 버전은 무엇인지 등을 여기서 설정합니다.

build.gradle (tour 프로젝트) 
// Gradle 플러그인 설정
plugins {
    id 'org.springframework.boot' version '3.5.11'  // Spring Boot 버전
    id 'io.spring.dependency-management' version '1.1.7'
    id 'java'
}

// 프로젝트 기본 정보
group = 'com.example'
version = '0.0.1-SNAPSHOT'
java { sourceCompatibility = JavaVersion.VERSION_21 }  // Java 21 사용

// 의존성(라이브러리) 설정
dependencies {
    // Spring Boot Starters (자주 쓰이는 라이브러리 묶음)
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'  // DB 연동
    implementation 'org.springframework.boot:spring-boot-starter-web'          // 웹 MVC
    implementation 'org.springframework.boot:spring-boot-starter-thymeleaf'    // HTML 템플릿
    implementation 'org.springframework.boot:spring-boot-starter-validation'  // 입력값 검증

    // 데이터베이스 드라이버
    runtimeOnly 'org.postgresql:postgresql'  // PostgreSQL 연결

    // 편의 라이브러리
    compileOnly 'org.projectlombok:lombok'         // 코드 자동 생성
    annotationProcessor 'org.projectlombok:lombok'  // Lombok 처리기

    // 테스트
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}
의존성역할없으면?
starter-web웹 서버, MVC 기능 제공웹 앱 불가
starter-data-jpa데이터베이스 연동 (JPA/Hibernate)DB 조작 불가
starter-thymeleafHTML 템플릿 엔진동적 HTML 불가
starter-validation입력값 검증 (@Valid 등)유효성 검사 불가
postgresqlPostgreSQL DB 연결 드라이버DB 접속 불가
lombokgetter/setter 자동 생성수동으로 작성해야 함

⚙️ 5. application.yml 분석

application.yml은 앱의 설정 파일입니다. 서버 포트, 데이터베이스 주소, 각종 설정값을 여기에 적습니다.

application.yml (tour 프로젝트) 
# 서버 설정
server:
  port: 8084          # 앱이 실행될 포트 번호

# 앱 이름
spring:
  application:
    name: tour

  # 데이터베이스 연결 설정
  datasource:
    url: jdbc:postgresql://localhost:5432/cheongdo  # DB 주소
    username: postgres                                 # DB 아이디
    password: 1004                                   # DB 비밀번호

  # JPA (데이터베이스 처리) 설정
  jpa:
    hibernate:
      ddl-auto: create   # 앱 시작 시 테이블을 새로 만듦
    show-sql: true       # 실행되는 SQL을 콘솔에 출력

  # Thymeleaf 설정
  thymeleaf:
    cache: false         # 개발 중 캐시 비활성화 (변경사항 즉시 반영)

# 파일 업로드 설정
file:
  upload-dir: C:/upload/
ddl-auto: create

위험! 앱 실행 시마다 DB 테이블을 모두 삭제하고 새로 만듭니다. 개발 초기에만 사용하고, 실제 서비스에서는 절대 사용하면 안 됩니다!

ddl-auto: update

기존 테이블은 유지하고, 변경된 부분만 업데이트합니다. Demo 프로젝트에서 사용. 기존 데이터가 보존됩니다.

ddl-auto: none

JPA가 DDL을 전혀 실행하지 않습니다. 실제 서비스(프로덕션) 환경에서 권장하는 설정입니다.

show-sql: true

JPA가 실행하는 SQL 쿼리를 콘솔에 출력합니다. 어떤 쿼리가 실행되는지 확인할 때 유용합니다.

⚠️

YAML vs Properties

application.yml과 application.properties는 같은 역할을 합니다.
YAML은 들여쓰기로 계층 구조를 표현하므로 더 보기 좋습니다.
들여쓰기(스페이스 2칸)가 매우 중요합니다. 탭 대신 스페이스를 사용해야 합니다!

🏛️ 6. MVC 패턴이란?

Spring Boot는 MVC (Model-View-Controller) 패턴을 사용합니다. 역할을 세 부분으로 나누어 코드를 구조화합니다.

역할클래스담당 업무예시
Model
(모델)		 Entity, Service, Repository 데이터 처리, DB 연동 Tour.java, TourService.java
View
(뷰)		 Thymeleaf HTML 화면 표시 tour/list.html
Controller
(컨트롤러)		 @Controller 클래스 요청 받아 처리 후 뷰에 전달 TourController.java
요청 처리 흐름 예시 
// 사용자가 /tour/list 주소로 접속하면...

// 1. Controller가 요청을 받음
@GetMapping("/list")
public String list(Model model) {
    // 2. Service를 통해 데이터 조회
    model.addAttribute("list", tourService.findAll());
    // 3. View(HTML) 이름 반환
    return "tour/list";  // templates/tour/list.html 파일을 렌더링
}

✅ 7. 챕터 핵심 요약

  • Spring Boot = Java 웹 애플리케이션을 쉽게 만드는 프레임워크
  • @SpringBootApplication = ComponentScan + AutoConfiguration + Configuration 합본
  • SpringApplication.run()으로 앱 실행 (내장 톰캣 시작)
  • build.gradle: 사용할 라이브러리(의존성)를 선언
  • application.yml: DB 연결, 서버 포트 등 설정값
  • MVC 패턴: Controller(요청처리) → Service(비즈니스) → Repository(DB) → View(화면)
  • ddl-auto: create는 개발용, 실제 서비스에서는 절대 사용 금지!
