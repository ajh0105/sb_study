🖥️ Thymeleaf 템플릿 엔진

Spring Boot의 기본 HTML 템플릿 엔진, Thymeleaf의 주요 속성을 실제 코드로 배웁니다. th:text, th:href, th:if, th:each 완전 정복!

⏱️ 예상 학습 시간: 45분 📝 핵심 속성: 10개+ 🎯 퀴즈: 50문제

❓ 1. Thymeleaf란?

Thymeleaf는 Spring Boot의 기본 HTML 템플릿 엔진입니다. Java에서 처리한 데이터를 HTML에 동적으로 표시할 수 있습니다.

📁

파일 위치

모든 Thymeleaf HTML 파일은 src/main/resources/templates/ 폴더에 위치합니다.
Controller에서 return "tour/list"를 반환하면 templates/tour/list.html 파일을 렌더링합니다.

📝 2. 핵심 속성 (th:*)

속성역할예시
th:text텍스트 표시<h1 th:text="${tour.name}"></h1>
th:href링크 URL 동적 생성<a th:href="@{/tour/detail/{id}(id=${tour.id})}">
th:src이미지 경로 동적 설정<img th:src="@{/images/{img}(img=${tour.thumb})}">
th:action폼 전송 URL 설정<form th:action="@{/tour/new}" method="post">
th:object폼 바인딩 객체 지정<form th:object="${tourFormDto}">
th:field폼 필드 바인딩<input th:field="*{name}">
th:if조건부 표시<div th:if="${loginMember != null}">
th:unless조건 반대 (if not)<div th:unless="${loginMember != null}">
th:each반복<tr th:each="tour : ${list}">
th:switchswitch 조건<div th:switch="${user.role}">

🔤 3. th:text & th:href

텍스트 & 링크 예시
<!-- th:text: 텍스트 출력 -->
<h2 th:text="${tour.name}">기본값(Thymeleaf 처리 전)</h2>
<!-- 결과: <h2>청도 와인 터널</h2> -->

<!-- 문자열 합치기 -->
<p th:text="|평점: ${tour.avgRating}점|"></p>
<!-- 결과: <p>평점: 4.5점</p> -->

<!-- th:href: URL 생성 -->
<!-- @{경로(파라미터=값)} 형식 -->
<a th:href="@{/tour/detail/{id}(id=${tour.id})}">상세보기</a>
<!-- 결과: <a href="/tour/detail/3">상세보기</a> -->

<!-- 이미지 -->
<img th:src="@{/images/{img}(img=${tour.thumb})}" alt="썸네일" />

🔁 4. th:each — 반복문

반복문 예시 (목록 출력)
<!-- th:each="변수 : ${컬렉션}" -->
<table>
  <thead>
    <tr><th>이름</th><th>카테고리</th><th>평점</th></tr>
  </thead>
  <tbody>
    <tr th:each="tour : ${list}">   <!-- list의 각 항목을 tour 변수에 -->
      <td th:text="${tour.name}"></td>
      <td th:text="${tour.category}"></td>
      <td th:text="${tour.avgRating}"></td>
    </tr>
  </tbody>
</table>

<!-- 반복 상태 변수 활용 -->
<tr th:each="tour, stat : ${list}">
  <td th:text="${stat.index}"></td>   <!-- 0, 1, 2... -->
  <td th:text="${stat.count}"></td>   <!-- 1, 2, 3... -->
  <td th:text="${stat.last}"></td>    <!-- 마지막이면 true -->
</tr>

❓ 5. th:if / th:unless — 조건문

조건문 예시
<!-- 로그인 상태에 따라 다른 UI 표시 -->
<div th:if="${session.loginMember != null}">
  <span th:text="${session.loginMember.name}"></span>님, 환영합니다!
  <a href="/member/logout">로그아웃</a>
</div>

<div th:unless="${session.loginMember != null}">
  <a href="/member/login">로그인</a>
  <a href="/member/new">회원가입</a>
</div>

<!-- 관리자만 보이는 버튼 -->
<a th:if="${session.loginMember?.role?.name() == 'ADMIN'}"
   th:href="@{/tour/new}">관광지 등록</a>

<!-- null 안전 접근: ?. 사용 -->
<p th:text="${user?.name ?: '비회원'}"></p>

📋 6. 폼(Form) 처리

Form 예시
<!-- th:object로 폼 객체 지정 -->
<form th:action="@{/member/new}" method="post"
      th:object="${memberFormDto}">

  <!-- th:field로 필드 자동 바인딩 -->
  <!-- name, id, value 속성이 자동 설정됨 -->
  <input type="text" th:field="*{username}"
         placeholder="아이디 입력" />

  <input type="password" th:field="*{password}" />

  <!-- 검증 오류 메시지 표시 -->
  <p th:if="${#fields.hasErrors('username')}"
     th:errors="*{username}"
     style="color:red">
  </p>

  <!-- select box -->
  <select th:field="*{category}">
    <option th:each="cat : ${categories}"
            th:value="${cat}"
            th:text="${cat}">
    </option>
  </select>

  <button type="submit">등록</button>
</form>
💡

th:field vs th:value 차이

th:field="*{name}": name, id, value 속성이 모두 자동 설정됨. 검증 오류도 처리됨
th:value="${dto.name}": value 속성만 설정됨
폼 처리 시에는 th:field를 사용하는 것이 올바른 방법입니다!

🔐 7. 세션 접근 & CSRF

세션 및 CSRF 처리
<!-- 세션 변수 접근 -->
<span th:text="${session.loginMember.name}"></span>

<!-- 모델 변수 접근 -->
<p th:text="${errorMessage}"></p>

<!-- 조건부 CSS 클래스 추가 -->
<li th:class="${tour.category == 'FOOD'} ? 'active' : ''">

<!-- URL 파라미터 여러 개 -->
<a th:href="@{/review/edit/{id}(id=${review.id}, tourId=${tour.id})}">
  수정
</a>
<!-- 결과: /review/edit/5?tourId=3 -->

✅ 8. 챕터 핵심 요약

  • Thymeleaf HTML은 templates/ 폴더에 위치
  • th:text="${변수}": 텍스트 출력
  • th:href="@{/path(param=${값})}": 동적 URL 생성
  • th:each="item : ${list}": 반복문
  • th:if="${조건}": 조건부 표시
  • th:object="${dto}" + th:field="*{field}": 폼 바인딩
  • th:errors="*{field}": 검증 오류 메시지 표시
  • ${session.loginMember}: 세션 변수 접근
