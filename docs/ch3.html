🔍 Repository

데이터베이스와 통신하는 계층입니다. SQL을 직접 작성하지 않아도 JpaRepository 인터페이스 하나로 CRUD가 완성됩니다!

⏱️ 예상 학습 시간: 35분 📝 핵심 어노테이션: 3개 🎯 퀴즈: 50문제

❓ 1. Repository란?

Repository는 데이터베이스 접근 계층입니다. Entity를 DB에 저장, 조회, 수정, 삭제하는 역할을 담당합니다.


Spring Data JPA의 JpaRepository 인터페이스를 상속하면, 기본적인 CRUD 메서드가 자동으로 만들어집니다!

Service
→
Repository
→
JPA/Hibernate
→
DB (SQL)

📚 2. JpaRepository 인터페이스

JpaRepository 기본 구조
// JpaRepository<엔티티 타입, 기본키 타입>
public interface TourRepository extends JpaRepository<Tour, Long> {
    // 구현 코드 없이 인터페이스만 선언해도 기본 메서드 자동 제공!
}
🎁

JpaRepository가 자동 제공하는 메서드들

  • save(entity) → INSERT 또는 UPDATE
  • findById(id) → SELECT WHERE id = ?
  • findAll() → SELECT * FROM table
  • deleteById(id) → DELETE WHERE id = ?
  • count() → SELECT COUNT(*)
  • existsById(id) → SELECT EXISTS WHERE id = ?

🏖️ 3. TourRepository 분석

TourRepository.java
public interface TourRepository extends JpaRepository<Tour, Long> {

    // 메서드명으로 쿼리 자동 생성!
    // "findBy" + "Category" = "WHERE category = ?" 쿼리 자동 실행
    List<Tour> findByCategory(TourCategory category);
}
✨

메서드명 쿼리 생성 규칙

Spring Data JPA는 메서드 이름을 분석하여 자동으로 SQL을 생성합니다!

  • findByName(String name) → WHERE name = ?
  • findByNameAndCategory() → WHERE name = ? AND category = ?
  • findByNameContaining() → WHERE name LIKE '%?%'
  • findAllByOrderByIdDesc() → ORDER BY id DESC
  • countByBoardType() → SELECT COUNT(*) WHERE board_type = ?

👤 4. MemberRepository 분석

MemberRepository.java
public interface MemberRepository extends JpaRepository<Member, Long> {

    // username으로 회원 조회 (로그인 시 사용)
    // Optional = 결과가 없을 수도 있음 (null 안전)
    Optional<Member> findByUsername(String username);

    // username 중복 확인 (AJAX)
    boolean existsByUsername(String username);
}
Optional<T>

결과가 있을 수도, 없을 수도 있는 값을 감싸는 컨테이너입니다. null을 직접 다루는 것보다 안전합니다.
.orElseThrow(): 없으면 예외 발생
.isPresent(): 존재 여부 확인

boolean existsBy~()

특정 조건의 데이터 존재 여부를 확인합니다. existsByUsername("홍길동")은 SELECT EXISTS(SELECT 1 FROM member WHERE username=?) 쿼리를 실행합니다.

⭐ 5. ReviewRepository — @Query 분석

ReviewRepository.java
public interface ReviewRepository extends JpaRepository<Review, Long> {

    // tourId로 리뷰 목록 조회
    List<Review> findByTourId(Long tourId);

    // JPQL로 평균 별점 계산 (복잡한 쿼리는 @Query 사용)
    @Query("SELECT AVG(r.starRate) FROM Review r WHERE r.tour.id = :tourId")
    Double findAverageRating(@Param("tourId") Long tourId);

    // 별점 내림차순 정렬
    List<Review> findByTourIdOrderByStarRateDesc(Long tourId);
}
@Query

JPQL(Java Persistence Query Language)을 직접 작성합니다. 메서드명 쿼리로 표현하기 어려운 복잡한 쿼리를 작성할 때 사용합니다.

@Param("tourId")

JPQL의 :tourId 파라미터와 메서드 인수를 연결합니다. :paramName과 @Param("paramName")이 매칭됩니다.

📝

JPQL vs SQL

SQL: SELECT AVG(star_rate) FROM review WHERE tour_id = ? (DB 테이블명/컬럼명)
JPQL: SELECT AVG(r.starRate) FROM Review r WHERE r.tour.id = :tourId (Java 클래스명/필드명)

🛡️ 7. Optional 올바른 사용법

Optional 활용 예시
// ❌ 나쁜 예: null 비교 (오래된 방식)
Member member = memberRepository.findByUsername(username);
if (member == null) {
    throw new Exception("없음");
}

// ✅ 좋은 예: Optional 사용
Member member = memberRepository.findByUsername(username)
    .orElseThrow(() -> new IllegalArgumentException("존재하지 않는 아이디입니다."));

// Optional의 주요 메서드들:
Optional<Member> opt = memberRepository.findById(1L);

opt.isPresent();                  // 값이 있으면 true
opt.isEmpty();                     // 값이 없으면 true
opt.get();                          // 값 꺼냄 (없으면 예외!)
opt.orElseThrow();                  // 없으면 NoSuchElementException
opt.orElse(null);                   // 없으면 null 반환
opt.orElseGet(() -> new Member()); // 없으면 람다 실행

✅ 8. 챕터 핵심 요약

  • JpaRepository<엔티티, PK타입> 상속만 해도 CRUD 메서드 자동 생성
  • 메서드 이름 규칙: findBy필드명 → WHERE 필드 = ? 쿼리 자동 생성
  • Optional<T>: null-safe 결과 반환, .orElseThrow()로 예외 처리
  • @Query: JPQL로 복잡한 쿼리 직접 작성
  • @Param: JPQL 파라미터 바인딩
  • JPQL: SQL처럼 보이지만 테이블명 대신 클래스명, 컬럼명 대신 필드명 사용
