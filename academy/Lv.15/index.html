1) 심화 이론

Deep Explaining

대량 데이터는 조회 설계가 곧 성능입니다.

핵심 포인트
  • page/size/sort 계약
  • PageResponse 표준화
  • 검색 조건 DTO
  • 인덱스 고려

2) 코드 (라인별 이유)

코드
public PageResponse<BoardResponse> search(int page, int size, String keyword) {
    Pageable pageable = PageRequest.of(page, size, Sort.by(Sort.Direction.DESC, "id"));
    Page<Board> result = boardRepository.findByTitleContaining(keyword, pageable);
    return PageResponse.from(result.map(BoardResponse::from));
}

3) 심화 해설

해설
  • 0-base page 기준을 프런트와 맞추세요.
  • 정렬 파라미터는 화이트리스트 검증이 필요합니다.
패키지 대화

Controller는 파라미터 수집, Service는 Pageable 조립, Repository는 조회 수행.
