1) 심화 이론

Deep Explaining

API가 많아질수록 포맷 통일이 생산성을 좌우합니다.

핵심 포인트
  • 성공/실패 공통 포맷
  • code/message/data
  • 버전 충돌 대비
  • 문서화 연계

2) 코드 (라인별 이유)

코드
public record ApiResponse<T>(String code, String message, T data) {
    public static <T> ApiResponse<T> ok(T data) { return new ApiResponse<>("OK", "성공", data); }
}

@GetMapping("/{id}")
public ResponseEntity<ApiResponse<MemberResponse>> findOne(@PathVariable Long id) {
    return ResponseEntity.ok(ApiResponse.ok(memberService.findById(id)));
}

3) 심화 해설

해설
  • 공통 래퍼는 프런트 파싱 비용을 크게 줄입니다.
  • message와 code를 분리하면 UI와 로깅이 쉬워집니다.
패키지 대화

Controller가 Service 결과를 표준 래퍼로 감싸 API 계약을 고정합니다.
