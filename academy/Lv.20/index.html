1) 심화 이론

Deep Explaining

운영은 기능 완성의 마지막 단계입니다. 문제를 빨리 찾고 빨리 복구하는 체계가 핵심입니다.

핵심 포인트
  • 환경변수 시크릿
  • 프로필 분리
  • 헬스체크/메트릭
  • 롤백 전략

2) 코드 (라인별 이유)

코드
spring:
  profiles:
    active: prod
  datasource:
    url: ${DB_URL}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics

# Dockerfile
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY build/libs/*.jar app.jar
ENTRYPOINT ["java","-jar","app.jar","--spring.profiles.active=prod"]

3) 심화 해설

해설
  • 민감값 하드코딩 금지는 필수 보안 원칙입니다.
  • 배포 후 health check 실패 시 자동 롤백을 준비하세요.
패키지 대화

애플리케이션 코드와 인프라/모니터링이 함께 동작할 때 운영 품질이 완성됩니다.
