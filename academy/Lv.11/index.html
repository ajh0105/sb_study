1) 심화 이론

Deep Explaining

REST는 기술이 아니라 약속입니다. URL, HTTP 메서드, 상태코드를 일관되게 설계해야 팀 협업이 쉬워집니다.

핵심 포인트
  • 리소스 중심 URL
  • ResponseEntity 상태코드
  • @RequestBody/@PathVariable
  • 서비스 재사용

2) 코드 (라인별 이유)

코드
@RestController
@RequestMapping("/api/boards")
@RequiredArgsConstructor
public class BoardApiController {
    private final BoardService boardService;

    @PostMapping
    public ResponseEntity<BoardResponse> create(@RequestBody @Valid BoardCreateRequest req) {
        Long id = boardService.create(req);
        return ResponseEntity.status(HttpStatus.CREATED).body(boardService.findOne(id));
    }

    @GetMapping("/{id}")
    public ResponseEntity<BoardResponse> findOne(@PathVariable Long id) {
        return ResponseEntity.ok(boardService.findOne(id));
    }
}

3) 심화 해설

해설
  • POST 성공은 201이 의미상 정확합니다.
  • PathVariable은 리소스 식별자 전달용입니다.
  • 기존 서비스 재사용으로 비즈니스 중복을 피합니다.
패키지 대화

Controller 응답만 JSON으로 바뀌고 내부 서비스/리포지토리 구조는 유지됩니다.
