1) Config 기초: @Configuration과 @Bean

Deep Explaining

@Configuration은 "설정 클래스"임을 선언합니다. @Bean은 "스프링 컨테이너가 관리할 객체"를 등록합니다. 핵심은 한 번 만들고 여러 곳에서 주입받아 재사용하는 것입니다.

기초 코드 (라인별 주석) 
package com.example.lv7.config; // 설정 패키지

import org.springframework.context.annotation.Bean; // Bean 등록 어노테이션
import org.springframework.context.annotation.Configuration; // 설정 클래스 선언
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder; // 비밀번호 해시 구현체
import org.springframework.security.crypto.password.PasswordEncoder; // 암호화 인터페이스

@Configuration // 이 클래스가 설정 클래스임을 스프링에 알림
public class PasswordConfig {

    @Bean // 아래 메서드가 반환하는 객체를 컨테이너에 등록
    public PasswordEncoder passwordEncoder() { // Service가 주입받아 사용할 타입
        return new BCryptPasswordEncoder(); // 구현체 생성
    }
}
Granular Analysis: public PasswordEncoder passwordEncoder()
부품문법적 의미왜 필요한가?
public접근 제어자스프링 컨테이너가 메서드를 호출해 Bean 생성 가능해야 함
PasswordEncoder반환형(인터페이스)구현체를 숨기고 의존성 역전 원칙 적용
passwordEncoder메서드명/Bean 이름주입 시 기본 식별자로 사용 가능
()파라미터 없음외부 입력 없이 고정 규칙 도구 생성

2) Config 심화 1: 프로퍼티 분리와 타입 안전 바인딩

왜 심화가 필요한가?

파일 경로, 제한 용량, 토큰 만료시간 같은 값은 코드 하드코딩 대신 설정 파일로 분리해야 운영 환경(local/prod)에서 유연하게 바뀝니다.

application.yml + @ConfigurationProperties 
# application.yml
app:
  upload:
    dir: C:/upload/lv7 # 업로드 기본 경로
  security:
    token-expire-minutes: 30 # 토큰 만료 시간(예시)

package com.example.lv7.config;

import lombok.Getter;
import lombok.Setter;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.context.annotation.Configuration;

@Getter
@Setter
@Configuration // 스프링 빈 등록
@ConfigurationProperties(prefix = "app") // app.* 설정을 이 클래스 필드에 바인딩
public class AppProperties {
    private Upload upload = new Upload(); // 하위 설정 객체
    private Security security = new Security();

    @Getter @Setter
    public static class Upload {
        private String dir; // app.upload.dir
    }

    @Getter @Setter
    public static class Security {
        private int tokenExpireMinutes; // app.security.token-expire-minutes
    }
}
문법 해설
  • @ConfigurationProperties는 문자열 설정을 자바 객체로 안전하게 매핑합니다.
  • 하위 클래스로 묶으면 설정 구조가 yml과 1:1로 대응되어 읽기 쉬워집니다.
  • Service는 이 객체를 주입받아 경로/시간값을 하드코딩 없이 사용합니다.

3) Config 심화 2: WebMvcConfigurer로 웹 동작 확장

MVC 설정 확장 코드 
package com.example.lv7.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.ResourceHandlerRegistry;
import org.springframework.web.servlet.config.annotation.ViewControllerRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer { // MVC 동작 커스터마이징

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/").setViewName("index"); // 단순 URL -> 뷰 매핑
    }

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/uploads/**") // 웹 접근 경로
                .addResourceLocations("file:///C:/upload/lv7/"); // 실제 파일 시스템 경로
    }
}
왜 중요한가?
  • addViewControllers는 단순 페이지 라우팅을 Controller 코드 없이 처리해 중복 감소.
  • addResourceHandlers는 업로드 파일을 웹에서 접근 가능하게 연결.
  • Config에서 웹 정책을 중앙 관리하면 추후 유지보수가 쉬워집니다.

4) 어노테이션 사전 (Config 편)

동작 원리 + 사용 시점
  • @Configuration: 설정 클래스 선언. Bean 생성 메서드를 담는 컨테이너.
  • @Bean: 반환 객체를 스프링이 관리하도록 등록.
  • @ConfigurationProperties: yml/properties 값을 객체로 바인딩.
  • WebMvcConfigurer: MVC 기본 동작을 프로젝트 정책에 맞게 확장.

5) Lv.7 실습 체크리스트

완료 기준
  1. 왜 PasswordEncoder를 Config에서 Bean으로 등록하는지 설명할 수 있다.
  2. @ConfigurationProperties로 설정값을 타입 안전하게 바인딩할 수 있다.
  3. WebMvcConfigurer로 뷰/리소스 정책을 분리할 수 있다.
  4. Config가 Service/Controller와 어떻게 연결되는지 말로 설명할 수 있다.
다음 Lv.8도 같은 방식으로 시작하며, Service 기초+심화(비즈니스 로직 중심)로 이어집니다.
