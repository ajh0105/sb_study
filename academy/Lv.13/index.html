1) 심화 이론

Deep Explaining

보안은 컨트롤러 앞에서 동작하는 필터 체인의 개념을 이해해야 제대로 다룰 수 있습니다.

핵심 포인트
  • encode/matches 원칙
  • permitAll vs authenticated
  • 보호 URL 설계
  • 보안 실패 응답

2) 코드 (라인별 이유)

코드
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public PasswordEncoder passwordEncoder() { return new BCryptPasswordEncoder(); }

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/login", "/api/members").permitAll()
                .anyRequest().authenticated())
            .httpBasic(Customizer.withDefaults());
        return http.build();
    }
}

3) 심화 해설

해설
  • 필터체인은 컨트롤러보다 먼저 실행됩니다.
  • 비밀번호는 절대 복호화하지 않고 matches로 비교합니다.
패키지 대화

SecurityConfig는 API 앞단 정책, Service는 도메인 규칙을 담당합니다.
