1) Repository란 무엇인가?

핵심 개념

Repository는 DB 작업 창구입니다. Service는 규칙을 생각하고, Repository는 "조회/저장"을 수행합니다. 즉 역할을 분리해 코드를 읽기 쉽게 만듭니다.

왜 JPA 상속을 먼저 배우나?
  • JpaRepository<Member, Long> 하나로 기본 CRUD를 즉시 확보할 수 있습니다.
  • 초보자는 SQL을 전부 직접 쓰기보다, "메서드 이름 -> 쿼리 생성" 규칙부터 익히는 것이 학습 효율이 높습니다.

2) 기본 JPA 상속 코드 (라인별 이유)

코드 
package com.example.lv2.repository; // repository 패키지 위치 선언

import com.example.lv2.entity.Member; // 어떤 엔티티를 다룰지 지정하기 위해 필요
import org.springframework.data.jpa.repository.JpaRepository; // JPA 기본 CRUD 기능 상속

public interface MemberRepository extends JpaRepository<Member, Long> { // Member 엔티티, PK 타입 Long
    // 아직 메서드가 없어도 save/findById/findAll/deleteById를 바로 사용 가능
}
문법 해설
코드 조각 뜻 왜 필요한가?
interface 구현 규약 선언 Spring Data JPA가 런타임에 구현체를 자동 생성
JpaRepository<Member, Long> 제네릭 타입 지정 어떤 엔티티와 PK를 다루는지 프레임워크가 정확히 인식
extends 상속 검증된 기본 기능을 재사용해 개발 속도/안정성 확보

3) Optional과 List를 추가해 기능 확장

확장 메서드 선언 
import java.util.List; // 여러 건 조회 결과 타입
import java.util.Optional; // 결과가 없을 수도 있을 때 안전한 타입

public interface MemberRepository extends JpaRepository<Member, Long> {

    Optional<Member> findByEmail(String email); // email은 보통 유니크, 결과 0개 또는 1개

    List<Member> findByEmailContaining(String keyword); // 검색 결과는 여러 개 가능

    List<Member> findByIdIn(List<Long> ids); // id 목록에 포함되는 여러 회원 조회

    boolean existsByEmail(String email); // 존재 여부만 빠르게 확인
}
Optional vs List 깊게 이해하기
타입 의미 실수 방지 포인트
Optional<Member> 값이 없을 수도 있음(0 or 1) null 직접 처리 대신 orElseThrow로 명확한 예외 흐름 구성
List<Member> 여러 건(0~N) 결과 없음은 isEmpty()로 명확하게 처리
Optional 사용 예 (Service에서 호출한다고 가정) 
Member member = memberRepository.findByEmail(email) // Optional 반환
    .orElseThrow(() -> new IllegalArgumentException("해당 이메일 회원이 없습니다.")); // 값 없으면 즉시 예외
List 사용 예 
List<Member> members = memberRepository.findByEmailContaining(keyword); // 여러 회원 조회
if (members.isEmpty()) {
    // 검색 결과 없음 처리
}

4) 어노테이션/문법 사전 (Lv.2 Repository 편)

필수 개념 정리
  • JpaRepository: CRUD + 페이징 + 정렬 기본 기능 제공.
  • Optional: null 처리 실수를 줄이는 컨테이너.
  • List: 다건 조회 표준 결과 타입.
  • findBy...Containing: SQL의 LIKE %keyword%와 유사한 검색 의미.
  • existsBy...: 데이터 전체를 가져오지 않고 존재 여부만 확인.

5) Lv.2 실습 체크리스트

완료 기준
  1. 왜 JpaRepository<Member, Long>를 상속하는지 설명할 수 있다.
  2. Optional을 써야 하는 경우와 List를 써야 하는 경우를 구분할 수 있다.
  3. 메서드 이름 기반 쿼리(findByEmail, findByEmailContaining)가 어떻게 동작하는지 설명할 수 있다.
