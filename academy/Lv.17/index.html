1) 심화 이론

Deep Explaining

조건이 많아질수록 QueryDSL이 유지보수와 타입 안정성에서 강력합니다.

핵심 포인트
  • Q타입 컴파일 검증
  • BooleanBuilder 조립
  • RepositoryCustom 분리

2) 코드 (라인별 이유)

코드
QBoard b = QBoard.board;
BooleanBuilder where = new BooleanBuilder();
if (cond.getKeyword() != null && !cond.getKeyword().isBlank()) {
    where.and(b.title.contains(cond.getKeyword()));
}
return queryFactory.selectFrom(b).where(where).fetch();

3) 심화 해설

해설
  • 동적 검색은 null/blank 처리 규칙을 통일하세요.
  • 검색 성능은 인덱스와 함께 검토해야 합니다.
패키지 대화

Service가 조건 DTO를 만들고 Custom Repository가 QueryDSL 실행을 담당합니다.
