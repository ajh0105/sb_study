1) 심화 이론

Deep Explaining

성능 문제는 대부분 쿼리 수와 트랜잭션 경계에서 발생합니다.

핵심 포인트
  • readOnly 적용
  • fetch join 활용
  • 쿼리 로그 기반 개선

2) 코드 (라인별 이유)

코드
@Transactional(readOnly = true)
public List<BoardResponse> findBoards() {
    List<Board> boards = boardRepository.findAllWithWriter();
    return boards.stream().map(BoardResponse::from).toList();
}

@Query("select b from Board b join fetch b.writer")
List<Board> findAllWithWriter();

3) 심화 해설

해설
  • N+1은 연관 조회 시 쿼리가 폭증하는 문제입니다.
  • 성능 최적화는 항상 측정값으로 검증해야 합니다.
패키지 대화

Service는 경계 선언, Repository는 최적화 쿼리 제공, DTO는 최소 응답 구성.
